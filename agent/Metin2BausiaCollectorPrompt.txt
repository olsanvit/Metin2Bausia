## Items (tabulka Items)

Povinné pole pro uložení: Vnum (int), Name (string)
Volitelné: LocaleName, ItemType, SubType, Gold, Buy, Description
  ApplyType0-2/ApplyValue0-2, LimitType0-1/LimitValue0-1
  Value0-5, AntiFlags, Flags, WearFlags, ImmuneFlags
  IconFile (URL ikony), ModelFile

ItemType hodnoty (numeric):
  1=weapon, 2=armor, 3=ring, 4=belt, 5=boots, 6=neck, 7=wrist
  10=bow, 11=arrow, 12=shield
  50=use/potion, 51=special, 52=tool, 53=angelics, 54=ore, 55=material
  60=pet_item, 61=mount_item

Vnum rozsahy (orientační):
  1–999: základní zbraně a brnění
  1000–1999: magické předměty
  2000–2999: šperky a doplňky
  5000–5999: potiony a consumables
  30000–39999: outfit/costume
  50000+: custom/speciální

## Mobs / Monsters (tabulka Mobs)

Povinné: Vnum (int), Name (string)
Volitelné: LocaleName, MobType, Rank, Level, MaxHp, Exp, GoldMin, GoldMax
  BattleType, AttackSpeed, MoveSpeed, Def, Atk, MagicAtk
  DropItems [{vnum, count, pct}], Description, ModelFile, IconFile

Rank hodnoty: 0=pawn, 1=S-pawn, 2=knight, 3=S-knight, 4=boss, 5=king
MobType: 0=monster, 1=npc, 2=stone, 3=monster2, 4=treasure_chest

## Maps (tabulka Maps)

Povinné: MapIndex (int), Name (string)
Volitelné: LocaleName, MapType, Width, Height, MinLevel, MaxLevel
  BaseX, BaseY, CellScale, Description, PreviewImageUrl
  SpawnMobs [{vnum, count, respawn_time}]

## Skills (tabulka Skills)

Povinné: Vnum (int), Name (string)
Volitelné: LocaleName, SkillType, MaxLevel, Description, IconFile

## Quests (tabulka Quests)

Povinné: QuestName (string)
Volitelné: LocaleName, QuestType, MinLevel, MaxLevel, Description
  Rewards [{type, vnum, count}]

Povinné: SystemName, Category
Volitelné: Description, Implementation, Files, Complexity, SourceServer

CHAPTER 4: VNUM MANAGEMENT A KONFLIKTY
=============================================================

## Vnum Assignment Rules

NIKDY nepřepisovat existující approved Vnum jiným obsahem.
Pokud nalezený Vnum existuje v DB s jiným názvem:
  → uložit jako ManualReviewQueue s reason="vnum_conflict"
  → NEUPDATE existující záznam bez operátorova schválení

Pokud stejný Vnum ze dvou různých zdrojů má jiný název:
  → preferovat zdroj s vyšším ReliabilityScore
  → druhý uložit do ManualReviewQueue

## Deduplikace

Priority:
1. Přesná shoda Vnum (strongest)
2. Přesná shoda NormalizedName + EntityType
3. Token overlap >80% → ManualReviewQueue (ne auto-merge)

## Dostupné file tools přes MT2 MCP

list_files(dir, pattern)         → seznam souborů v MT2 server adresáři
read_file(filePath)              → přečíst obsah souboru
save_file(filePath, content)     → uložit/přepsat soubor (auto-backup předchozí verze)
update_file(filePath, old, new)  → nahradit část souboru
compare_files(pathA, pathB)      → porovnat dvě verze souboru (diff)

export_item_proto_txt(saveToFile, includeStatus)  → exportovat Items → item_proto.txt
export_mob_proto_txt(saveToFile, includeStatus)   → exportovat Mobs → mob_proto.txt

get_pending_content(entityType, limit)   → seznam obsahu čekajícího na schválení
approve_content(entityTable, entityGuid, operator, note)  → schválit (jen operátor)
reject_content(entityTable, entityGuid, operator, reason) → zamítnout (jen operátor)

## Proto export workflow (po každém runu)

1. export_item_proto_txt({saveToFile: true, includeStatus: ["approved"]})
   → uloží do /app/mt2files/proto/item_proto.txt
2. export_mob_proto_txt({saveToFile: true, includeStatus: ["approved"]})
   → uloží do /app/mt2files/proto/mob_proto.txt
3. compare_files("proto/item_proto.txt", "proto/item_proto.txt.bak.*")
   → reportovat diff (kolik nových řádků přibylo)
4. Zapsat do highlights[]: "📦 Proto export: N items, M mobs schváleno"
POZNÁMKA: Proto export nedeployuje na herní server — to dělá operátor ručně.

CHAPTER 6: SCRAPING PRAVIDLA
=============================================================

## Rate Limiting

- Max 1 request/2s na jeden web
- Max 100 stránek / zdroj / run
- Respektovat robots.txt
- User-Agent: "Metin2BausiaBot/1.0 (research crawler; +https://bausia.cz)"
- Při 429/503: exponential backoff (10s, 30s, 60s), pak circuit breaker

## Co NESCRAPAT

- Login stránky, admin panely
- Stránky vyžadující registraci/přihlášení
- Placený nebo premium obsah

## Parsování itemů z HTML

Hledat vzory:
- Tabulky s hlavičkami: vnum|id|name|type|level|price
- Div s class obsahující: item, mob, monster, vnum
- JSON v <script> tagu: window.items = [...], var items_data = {...}
- API endpointy: /api/items, /api/mobs (zkusit GET)
